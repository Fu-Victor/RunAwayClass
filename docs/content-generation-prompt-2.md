# 逃课模拟器 - 文案生成任务（第二批）

请按以下要求，将三个 YAML 配置文件中的 [TODO] 占位符替换为正式文案。

## 配置文件位置

- src/content/courseNames.yaml —— 课程名池
- src/content/teacherNames.yaml —— 老师名池
- src/content/feedbackTexts.yaml —— 数值区间反馈文案

## 文案总则

1. 搞笑优先：轻松幽默，善用网络梗、大学生黑话、抽象文化
2. 绝不严肃：不要出现说教、鸡汤、正式公告口吻
3. 篇幅紧凑：课程名 20 字以内，老师名 4 字以内，反馈文案 20 字以内

## 1. courseNames.yaml —— 课程名池

格式为纯字符串数组。要求：
- 总数 30~40 条
- 命名模式任选其一或混用：
  a) 真实课程 + 括号搞笑注释（如"高等数学（睡觉版）"）
  b) 纯虚构搞笑课名（如"摸鱼学导论"）
  c) 大学生活场景化命名（如"食堂鉴赏与实战"）
- 方向参考：水课、硬课、体育课、选修课、实验课、网课、实习相关
- 每条保留方括号 [TODO] 的为占位，替换掉整个字符串（包括 [TODO] 前缀）

## 2. teacherNames.yaml —— 老师名池

两组，各 10 条以上。格式为"姓氏 + 双字特征名"：
- roll_call_lover 组：名字体现"严厉、点名、查岗、记过"等特征（如"李点名""张严打"）
- roll_call_hater 组：名字体现"划水、随缘、放水、不管"等特征（如"王划水""赵佛系"）

## 3. feedbackTexts.yaml —— 数值区间反馈文案

四个维度，每个维度若干区间。只替换 text 字段，不要修改 range 和 level。
- mood（心情）：5 个区间，状态越差语气越丧越搞笑
- energy（精力）：4 个区间，状态越差越夸张
- hunger（饱腹）：4 个区间，从"肚子饱饱"到"饿到幻觉"
- entertainment（娱乐）：4 个区间，从"精神丰富"到"精神枯竭"

文案应随 level 增大而逐级夸张，如 level 0 的描述偏正面/平淡，level 最高的描述接近崩溃/绝望但保持搞笑。

## 输出要求

直接修改三个 YAML 文件，将 [TODO] 替换为正式文案，保留文件原有的注释和结构。修改完成后运行 npx tsc --noEmit 确认无类型错误。
