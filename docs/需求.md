# 《逃课模拟器》游戏需求文档

> 版本：v1.0 | 最后更新：2026-05-30
> 本文档仅描述游戏流程需求与画面需求，不涉及技术选型。

---

## 一、游戏概述

**游戏类型：** 文字决策 + 肉鸽策略，网页端单机游戏

**游戏风格：** 搞笑、无厘头。文案保持轻松幽默、避免过于严肃的内容。

**核心玩法：** 玩家扮演一名大学生，在 7 天的校园生活中，每晚对次日的课程做出行为决策（上课 / 旷课 / 帮人代课 / 找人代课），同时应对随机突发事件，维持各项数值不崩盘，最终活过第 7 天即可通关。通关时的数值表现决定结算评价。

**单局时长：** 约 15-25 分钟。

---

## 二、核心游戏流程

### 2.1 游戏主循环

```
游戏开始
  │
  ├── [夜晚决策阶段] ←──────────────────────────┐
  │   ├── 查看次日课表（4节课）                    │
  │   ├── 为每节课选择行为                         │
  │   └── 决定今晚行为（熬夜/娱乐/早睡等）           │
  │       │                                        │
  │       ▼                                        │
  │   [白天推进阶段]                                │
  │   ├── 按时间顺序逐节推进课程                     │
  │   ├── 课间可能触发随机事件                       │
  │   └── 处理事件中的决策选项                       │
  │       │                                        │
  │       ▼                                        │
  │   [每日结算阶段]                                │
  │   ├── 结算当日所有数值变动                       │
  │   ├── 检测是否触发失败条件                       │
  │   ├── 检测是否触发学分预警                       │
  │   └── 若未失败且未到第7天 → 进入下一夜 ──────────┘
  │       │
  │       ▼
  │   [第7天结算 → 通关评价]
  │   └── 根据最终数值给出搞笑评级
  │
游戏结束
```

### 2.2 各阶段详细说明

#### 2.2.1 夜晚决策阶段

每夜展示内容：

- **次日课表**：4 节课的信息卡片，每张卡片显示：
  - 课程名称（随机生成）
  - 课程属性（水课 / 正课）
  - 任课老师名称（随机生成）
  - 老师属性（爱点名 / 不爱点名）
  - 上课时间（如"8:00-10:00"）
  - 当前估算点名概率（基于课程属性+老师属性+时间段综合计算）

- **每节课的行为选项**：
  - **老实上课**：基础获得学分，正常消耗精力/饥饿
  - **旷课跑路**：不获得学分，有概率被点名扣学分，但可自由支配该时段（补觉/吃饭/娱乐）
  - **帮人代课**：帮别人上课赚外快（需已解锁），获得金钱，消耗精力
  - **找人代课**：花钱找人替自己上课（需已解锁），消耗金钱，规避点名风险，但可能有被坑的风险

- **今晚行为选项**（每晚必选一项）：
  - **早睡养生**：精力大幅恢复，舍友好感度微升
  - **熬夜打游戏**：娱乐值上升，精力下降，舍友好感度下降
  - **通宵赶作业**：学分微升，精力大降
  - **出去浪**：娱乐大幅上升，心情上升，金钱消耗，舍友好感度变化（正向或负向随机）
  - **正常休息**（默认）：各项数值小幅波动

#### 2.2.2 白天推进阶段

按课程时间顺序（4个时段）自动推进，每节课结算一次行为结果：

1. 展示该节课的行为结果动画（如"你偷偷从后门溜了" / "老师正在激情点名中…"）
2. 课间可能触发 **随机事件**（详见 2.3），玩家需做出选择
3. 处理完当前课程及事件后，进入下一节

#### 2.2.3 每日结算阶段

每天结束时统一结算：

1. 汇总当日所有数值变化，播放结算动画
2. 检测各项数值是否触发失败条件（见 3.2）
3. 检测学分是否低于预警值，若是则弹出搞笑警告
4. 展示"第 X 天结束"过渡画面，进入下一夜

---

### 2.3 随机事件系统

随机事件在**课间**触发，触发概率受当天课程属性、玩家当前状态等综合影响。事件以**弹窗/动画**形式呈现，玩家需在 2-3 个选项中做出选择。

**事件类型示例：**

| 事件类型 | 触发条件 | 示例文案 | 选项影响 |
|---------|---------|---------|---------|
| 点名危机 | 老师爱点名 + 玩家选了旷课 | "舍友紧急来电：'兄弟！下节课老李要查人！！'" | A. 飞奔回教室（精力-） B. 不管了爱咋咋地（大概率扣学分） C. 叫舍友帮忙喊到（消耗舍友好感度） |
| 课堂提问 | 玩家在上课 + 老师随机点名 | "老师推了推眼镜：'这位同学，你来回答一下'" | A. 自信作答（学分微+） B. 低头装死（有概率被记） |
| 社交邀约 | 舍友好感度较高 | "舍友发来微信：'下课去食堂整点？'" | A. 必须去（饮食+ 心情+ 金钱-） B. 拒绝（舍友好感度-） |
| 老师私信 | 随机 | "某老师发来微信：'同学，你上次作业是不是没交？'" | A. 立马补交（精力- 学分保底） B. 装没看见（可能扣学分） |
| 天降横财 | 随机低概率 | "你在教学楼门口捡到了 ¥50！" | A. 揣兜里（金钱+） B. 交给失物招领（心情微+，无事发生） |
| 突击测验 | 正课 + 玩家多日未上课 | "据可靠消息，下节课有突击测验！！" | A. 借舍友笔记狂补（精力- 学分微+） B. 相信自己裸考实力（高风险高回报） |

**事件设计要求：**
- 每个事件至少 2 个选项，最多 3 个
- 每个选项的影响不能完全优于另一个，必须有权衡
- 文案必须搞笑，可玩梗、整活
- 同一局内事件尽量不重复

## 三、数值系统

### 3.1 数值一览

#### 核心数值（直接影响通关/失败）

| 数值名称 | 说明 | 范围 |
|---------|------|------|
| **学分** | 核心目标值，上课/帮人代课等获取 | 0 ~ 100+ |
| **心情** | 核心存活值，归零即失败 | 0 ~ 100 |

#### 次要数值（影响心情及其他系统）

| 数值名称 | 说明 | 范围 |
|---------|------|------|
| **精力** | 代表疲劳度，低精力增加上课睡着概率 | 0 ~ 100 |
| **饱腹** | 代表饥饿度，低饱腹持续扣心情 | 0 ~ 100 |
| **娱乐** | 代表精神满足度，低娱乐影响上课效率 | 0 ~ 100 |

#### 辅助数值（影响事件与解锁）

| 数值名称 | 说明 | 范围 |
|---------|------|------|
| **金钱** | 用于购买道具、找人代课等，次要系统 | 0 ~ N |
| **舍友好感度** | 影响旷课提醒、社交事件等 | 0 ~ 100 |

### 3.2 数值机制

#### 学分
- **通关值**（阈值A）：第 7 天结算时学分 ≥ 此值方可通关
- **预警值**（阈值B）：当学分低于此值时触发警告提示（B > A，即提前预警）
- **家教值**（阈值C）：当学分高于此值时，解锁 **"家教"行为**（可在非上课时段选择做家教赚取金钱，同时小幅提升学分）
- 学分主要来源：老实上课、帮人代课、部分正面事件
- 学分主要扣减：旷课被点名、找人代课被坑、部分负面事件
- 心情会影响学分获取效率：
  - 心情较高时：每次上课获得额外学分（学霸附体 buff）
  - 心情较低时：每次上课获得学分衰减（人在教室魂在飞 debuff）

#### 心情
- **崩溃值**（阈值D）：心情 ≤ D 时直接触发游戏失败
- 心情每日结算：精力、饱腹、娱乐三项的加权平均值与期望值比较，高于期望则心情上升，低于则下降
- 心情也受事件、决策直接影响
- 心情过低时触发特殊对话/动画反馈（"你已经麻了""生无可恋.jpg"等）

#### 精力
- 上课、帮人代课消耗精力
- 旷课可补觉恢复精力
- 精力过低时：上课有概率触发"睡着了"事件，扣学分
- 精力影响"今晚行为"中熬夜类选项的风险

#### 饱腹
- 随时间自然下降
- 旷课期间可选择吃饭恢复
- 饱腹过低时：持续扣除心情，触发"肚子咕咕叫"等搞笑提示

#### 娱乐
- 随时间自然缓慢下降
- 可通过今晚行为、特定事件恢复
- 娱乐过低时：上课有概率触发"狂刷手机"事件，扣学分；心情持续下降

#### 金钱（次要系统）
- 来源：帮人代课、家教（需解锁）、随机事件
- 用途：找人代课、购买道具（如有）、部分事件的付费选项
- 金钱不足时，涉及花费的选项将被锁定

#### 舍友好感度
- 影响：旷课时舍友是否愿意帮你打掩护（减少被点名概率）
- 来源：一起娱乐、保持良好作息等正面互动
- 扣减：熬夜行为影响舍友休息、拒绝舍友邀约等

### 3.3 难度分级

游戏开始时可选难度。所有玩家行为（上课/旷课/代课/家教等）在任何难度下均全量开放。难度仅影响数值设计：

| 难度维度 | 简单 | 普通 | 困难 |
|---------|------|------|------|
| 初始数值 | 各项较高 | 各项适中 | 各项较低 |
| 课程正课比例 | 正课少，水课多 | 均衡 | 正课多，水课少 |
| 老师点名概率 | 普遍不爱点名 | 正常分布 | 点名狂魔扎堆 |
| 随机事件频率 | 正面事件偏多 | 均衡 | 负面事件偏多 |
| 数值期望值要求 | 较低 | 正常 | 较高 |
| 行为收益 | 各项行为正面收益放大 | 正常 | 各项行为正面收益缩水，负面风险放大 |

## 四、课程与老师随机生成

### 4.1 课程属性

每局游戏开始时，随机生成 7 天共 28 节课（每天 4 节）：

- **课程名称**：从搞笑课名池中随机抽取（如"高等数学（睡觉版）""摸鱼学导论""食堂鉴赏与实战"等）
- **课程类型**：水课 / 正课（影响学分获取量和点名后果）
- **上课时间**：每天 4 个固定时段（如 8:00 / 10:00 / 14:00 / 16:00）

### 4.2 老师属性

每门课随机分配一位老师：

- **老师名称**：从搞笑老师名池中随机抽取（如"李点名""王划水""张严查"等）
- **点名倾向**：爱点名 / 不爱点名（影响点名概率）
- **特殊属性**（低概率附加）：如"随机提问狂魔""课件念经大师""签到只认手写"等，影响该节课的特殊事件池

### 4.3 点名概率计算

每节课的实际点名概率由以下因子综合决定：

- 老师点名倾向（权重最高）
- 课程类型（正课更易点名）
- 上课时间段（早八时段点名概率偏高）
- 玩家历史旷课记录（旷课越多，被盯上的概率越大）

该概率在夜晚决策时以 **估算值** 形式展示给玩家（如"点名概率：低 / 中 / 高 / 极高"），实际点名结果在白天推进时结算。

## 五、画面与 UI 需求

### 5.1 整体布局

游戏采用**三栏仪表盘布局**，背景为深色/主题色，模拟一个"大学生活监控中心"的视觉感。

```
┌──────────────┬──────────────────────────────┬──────────────┐
│              │                              │              │
│   模拟手机    │       中央展示区              │   状态栏      │
│   (左侧)     │       (核心动画/事件区)        │   (右侧)     │
│              │                              │              │
│  ┌────────┐  │   ┌──────────────────┐       │  学分: 42    │
│  │ 微信   │  │   │                  │       │  心情: ████  │
│  │ (消息) │  │   │   角色动画/      │       │  精力: ██░░  │
│  ├────────┤  │   │   事件弹窗/      │       │  饱腹: ███░  │
│  │ 课表   │  │   │   心理描述       │       │  娱乐: █░░░  │
│  │ (课程) │  │   │                  │       │  金钱: ¥28   │
│  └────────┘  │   └──────────────────┘       │  舍友: ♥♥♡  │
│              │                              │              │
└──────────────┴──────────────────────────────┴──────────────┘
```

### 5.2 左侧：模拟手机

一个手机外框的视觉组件，内含两个可切换的 App 界面：

#### 微信界面
- 聊天列表样式，显示最近的偶发消息
- 消息类型：舍友群聊、老师私信、班级通知、神秘匿名消息等
- 点击消息可查看详情/触发对应事件
- 未读消息有红点提示
- 消息内容以搞笑聊天记录形式呈现（可带表情包文字描述）

#### 课表界面
- 表格/卡片样式，展示最近几日的课程安排
- 每节课显示：课程名、老师名、课程类型标签、点名概率标签
- 当天已过的课程灰色显示，即将到来的高亮
- 可左右滑动查看不同天的课表

### 5.3 右侧：状态栏

- 纵向排列的数值面板
- 每个数值以 **图标 + 名称 + 数值/进度条** 形式展示
- 数值变化时有**颜色闪烁动画**（绿色上升 / 红色下降）
- 达到危险阈值时进度条变为警告色（橙/红），并伴随抖动
- 可考虑加入搞笑的状态描述文字（如心情=10时显示"emo中，勿扰"）

### 5.4 中央：展示与交互区

这是游戏的核心视觉区域，根据不同阶段展示不同内容：

#### 夜晚决策时
- 横向排列 4 张课程卡片，卡片上展示课程信息
- 每张卡片下方有行为选择按钮组
- 底部为今晚行为选择区
- 选完后有"确认提交"按钮，按钮文案需搞笑（如"开摆！""冲！""明天再说"等）

#### 白天推进时
- 当前课程结果动画展示（如一个角色小人做出对应动作的简易动画）
- 配合文字描述当前发生的事（"你正在教室后排摸鱼…"）
- 事件触发时弹出决策弹窗，选项以醒目按钮呈现

#### 每日结算时
- 数值变动汇总展示（增减数值飞入动画）
- 一天结束的过渡文字（如"第3天结束，你还活着，真是奇迹"）

### 5.5 动画与反馈要求

由于画面以文字和按钮为主，动画反馈是提升体验的关键：

- **数值变化动画**：数值上升/下降时有明显的 +N / -N 飞字特效，配合绿色/红色
- **状态特效**：
  - 心情极低时：屏幕边缘泛灰/画面微微颤抖
  - 被点名时：屏幕短暂红闪 + 震动
  - 数值达阈值时：弹窗动画（如学分突破家教值时弹出"你已经是学神了，开班授课吧！"）
- **过渡动画**：阶段切换时有流畅的过渡效果（淡入淡出、滑动等）
- **趣味反馈**：
  - 连续旷课多天后：中间区域出现"通缉令"风格的搞笑画面
  - 数值全面崩盘时：出现"摆烂认证"等搞笑徽章动画
  - 通关时：根据评级展示不同搞笑结局画面

## 六、随机文案与反馈需求

### 6.1 文案设计要求

- **全量随机池**：各类文案（课程名、老师名、事件描述、结算评语、数值变化提示等）均从预设池中随机抽取
- **绝不严肃**：所有文案必须轻松搞笑，善用网络梗、大学生黑话、抽象文化
- **上下文感知**：文案需根据当前数值区间、玩家行为历史做差异化展示

### 6.2 需要随机文案的场景

| 场景 | 文案需求 | 示例 |
|------|---------|------|
| 课程名生成 | 搞笑课名池，约 30+ 条 | "中国近现代睡梦史""Java 从入门到放弃""食堂菜品鉴赏与实测" |
| 老师名生成 | 搞笑老师名池，约 20+ 条 | "李点名""王划水""张挂科""赵念经" |
| 决策前心理描述 | 夜晚决策时中央区展示，基于当前状态 | 心情低时："你已经麻了，爱咋咋地"；学分高时："扶我起来，我还能卷" |
| 上课结果描述 | 每节课结束后的结果文字 | 上课："你认真听讲，但灵魂已经飞到了食堂"；旷课："你在宿舍的床上度过了一个安详的上午" |
| 事件文案 | 所有随机事件的标题和描述 | 见 2.3 事件表格 |
| 结算评语 | 每日结算和最终结算的评语 | 学分低："你离退学又近了一步"；通关："恭喜你以微弱优势成功毕业，学位证是打印店做的" |
| 数值阈值提示 | 数值达到特定区间时 | 精力<20%："眼皮已经在打架了，建议立刻找地方昏迷"；学分>家教值："你已经强到可以开辅导班了" |
| 结局评价 | 第7天结束后给出搞笑评级 | 根据最终学分给出称号，如"学渣之王""逃课仙人""卷王之王""侥幸毕业的幸运儿" |

### 6.3 数值区间差异化反馈

每当数值进入不同区间时，触发对应的描述变化（以心情为例）：

| 心情区间 | 状态描述 | 视觉反馈 |
|---------|---------|---------|
| 80-100 | "心情愉悦，感觉自己是天选之子" | 屏幕微微暖色调 |
| 50-79 | "心态平稳，问题不大" | 正常 |
| 20-49 | "开始烦躁，想打人" | 屏幕边缘微微灰色 |
| 5-19 | "emo中，生人勿近" | 画面颤抖，灰色加重 |
| 0-4 | "已崩溃，建议重开" | 失败前夕的红色闪烁 |

同理，精力、饱腹、娱乐也需设计对应的区间文案，此处不一一列举，但需在开发时固化。

## 七、通关与评价

### 7.1 通关条件
- 活过第 7 天（心情未归零，即未触发崩溃失败）
- 第 7 天结算时，学分 ≥ 通关值

### 7.2 失败条件
- **心情归零**：触发"崩溃退学"结局（唯一的失败条件）

### 7.3 结算评价

通关后根据**多维数值综合表现**给出结算评价，分为**评级**和**称号**两层：

- **评级**：由加权综合分决定，决定评价的档次（S / A / B / C / D）
- **称号**：在同一评级内，根据各项数值的高低偏向、玩家行为风格，从称号池中匹配最合适的称号

#### 综合评分维度

结算综合分由以下维度加权计算：

| 维度 | 说明 | 权重 |
|------|------|------|
| 最终学分 | 第7天结算时的学分值 | 高 |
| 心情均值 | 7天心情的平均水平 | 中 |
| 精力/饱腹/娱乐均值 | 三项次要数值的平均水平 | 中 |
| 金币余额 | 最终持有金钱数 | 低 |
| 舍友好感度 | 最终好感度值 | 低 |
| 旷课率 | 7天内旷课节数占比 | 影响评级偏移 |

#### 评级档次（示例）

| 评级 | 条件（综合分区间） | 基调 |
|------|------------------|------|
| S | 顶尖表现 | "你这一周过得比校长还滋润" |
| A | 优秀表现 | "稳中带皮，大学生活典范" |
| B | 中等表现 | "凑合过吧，还能离咋的" |
| C | 勉强通关 | "在退学边缘反复横跳，但就是不掉下去" |
| D | 擦线通关 | "教务处的退学通知书都印好了，又被你撕了" |

#### 称号匹配

同一评级下，根据玩家数值偏向和行为特征匹配不同称号。称号由**主标签** + **副标签**组合或从预设称号池中匹配：

**称号匹配示例（以评级 A 为例）：**

| 数值特征 | 称号 | 评语 |
|---------|------|------|
| 学分突出 + 心情良好 | "卷王之王" | "你不仅学得好，心态还稳，同学们怀疑你是 AI" |
| 学分突出 + 心情偏低 | "苦行學霸" | "学分到手了，但你的精神状态让舍友担忧" |
| 金币多 + 旷课率高 | "商业鬼才" | "课是一节没上，钱是一分没少挣，你是懂大学生存的" |
| 舍友好感度极高 | "社交恐怖分子" | "你的人缘好到舍友愿意替你去考试" |
| 各项均衡无短板 | "六边形战士" | "德智体美劳全面发展的当代好青年（稀有物种）" |
| 旷课率极高但学分勉强够 | "逃课仙人" | "你用实力证明了：课，是真的可以不上" |
| 代课次数多 | "金牌替身" | "你帮别人上的课比给自己上的还多，代课界的传奇" |

**称号匹配示例（以评级 C 为例）：**

| 数值特征 | 称号 | 评语 |
|---------|------|------|
| 学分擦线 + 心情尚可 | "摆烂の胜利" | "心态好就是真的好，学分是什么？不重要" |
| 各项全面拉胯 | "侥幸毕业" | "学位证是打印店 50 块做的，但好歹是个证" |
| 金币多但其他差 | "暴发户学渣" | "有钱能使鬼推磨，你硬是靠钞能力撑过了这周" |
| 靠舍友硬抬 | "寄生虫の奇迹" | "如果没有舍友，你第一天就退学了" |

#### 失败时的评价（心情归零）

即使失败也给出搞笑评价，不让人沮丧：

| 失败原因 | 称号 | 评语 |
|---------|------|------|
| 心情归零 | "精神崩坏" | "你的心理防线比纸还薄，建议下学期选修《心态管理》" |
| 心情归零（学分极低） | "全面溃败" | "学习和生活双双缴械，你可能是来大学旅游的" |
| 心情归零（学分尚可） | "高处不胜寒" | "学分够了，人没了。下次记得对自己好一点" |

#### 称号设计原则
- 每个评级至少匹配 5 种不同称号，确保重复游玩的新鲜感
- 称号倾向由 2-3 个最突出的数值特征共同决定
- 评语文案保持搞笑、玩梗，绝不严肃说教
- 同一评级内出现多个匹配时，优先选择更"贴标签"的称号（即与玩家行为特征反差最大的）

## 八、附录：关键词速查

| 关键词 | 含义 |
|-------|------|
| 通关值 | 学分的最低过关线，第7天结算时不得低于此值 |
| 家教值 | 学分高于此值时解锁"家教"行为 |
| 预警值 | 学分低于此值时触发警告提示 |
| 崩溃值 | 心情低于此值时立即失败 |
| 水课 | 容易混的课，学分低，点名概率低 |
| 正课 | 硬核课程，学分高，点名概率高 |
| 代课 | 替别人上课赚外快，或找人替自己上课 |
| 好感度 | 舍友好感度，影响旷课掩护和社交事件 |
